## M1.5c implementation notes

- `NavBar` is rendered per-page rather than via a layout-level store. Pages render `<NavBar ... />` then content; the `(app)` layout is a paper-bg passthrough and the trip `[slug]` layout is a bare `{@render children()}`. Avoids the snippet-in-store gotcha and keeps NavBar config colocated with the page that owns the data.
- `<TripTabs>` is a separate component used by the three top-level trip pages (overview, phases list, settings). Sticky at `top-[57px]` so it docks under the NavBar (which is `top-0`).
- `<PhaseColorPicker>` writes a hidden-styled radio group with `name="color"`. Storage stays plain hex text (no schema change). Server-side actions still accept any color string — the picker is the only client guard.
- `phasePalette` lives in `src/lib/utils/phase-palette.ts`, not the `.svelte` file, because Svelte 5 doesn't allow `export const` from a script tag.
- E2E heading expectation flipped from `'Trips'` to `'Waypoint'` since the trips-list page no longer renders an in-content `<h1>` — the NavBar is the page's only h1.

## M1.5b implementation notes

- `Pill`: variant + size per handoff (`booked|pending|planned|trip|info|default|ink`, `sm|md`). Pending/info use static hex bgs since gold-tint/sky-tint aren't tokenized; promote to `@theme` if M3 needs them programmatically.
- `Card`: `href`, `onclick`, or static — same shell three ways. `accent` takes a CSS color string (e.g. a phase color hex), rendered as a 4px left border via inline style.
- `Button`: `primary` is ink/paper, `moss` is the green CTA, `ghost` is bordered, `soft` is surface-2. Optional `icon` snippet renders before children.
- `SectionH`: small-caps moss heading + optional `right` snippet (per handoff).
- `TypeIcon`: 6 inline SVGs keyed off `ItemType`. `sub` is accepted in the props (frozen API) but not yet differentiated; sub-typed glyphs are a no-op until M6 fleshes out subtypes.
- `Avatar`: initial OR placeholder (dashed) OR img. Single-member use only — `AvatarStack` is deferred to M2 per spec.
- `NavBar`: `back` toggles a chevron (uses `backHref` if provided, else `history.back()`). `right` snippet is the only slot for actions.
- `FAB`: fixed bottom-right ink circle. Default plus icon, override via `icon` snippet.
